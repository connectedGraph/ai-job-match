# 智能人岗匹配与数据治理系统 - 使用清单 (Checklist & Manual)

本使用清单旨在帮助您理清项目拷贝后的执行流程、环境配置方法、以及核心功能的使用核对。

## 📅 第一阶段：环境与数据核对

### 1. 依赖安装
* [ ] **Python 环境**：确保 Python 3.10+，并在根目录下运行以下命令安装所需依赖库：
  ```powershell
  pip install -r requirements.txt
  ```
  *(对应配置文件：[requirements.txt](file:///D:/1/CS&AI/个人项目/ALL/job_system - 交付版本/requirements.txt))*
* [ ] **前端环境**：确保 Node.js v18+ 已就位，分别进入两个前端文件夹安装依赖包：
  ```powershell
  # 安装学生端前端
  cd career-planner/frontend
  npm install
  cd ../..

  # 安装管理端前端
  cd job-admin/frontend
  npm install
  cd ../..
  ```

### 2. 共享数据资产检查
* [ ] **数据覆盖**：已经通过拷贝将 `D:\1\CS&AI\个人项目\ALL\dataset` 目录覆盖至项目根目录下。
* [ ] **数据文件核对**：检查 [dataset/](file:///D:/1/CS&AI/个人项目/ALL/job_system - 交付版本/dataset) 目录中是否包含以下核心文件：
  * `career.json`：全量岗位画像主数据库（约 45.8 MB）。
  * `db/tag_center/`：存储了标准的标签主字典（`tag_master.json`）及向量检索索引文件。
  * `db/domain_center/`：行业领域字典数据。

### 3. 后端路径适配验证
* [ ] **后台批处理目录解耦**：系统已完成对后台 Batch 处理及 JD 结构化 Pipeline 的目录解耦。
  * 管理端后端的 [project_paths.py](file:///D:/1/CS&AI/个人项目/ALL/job_system - 交付版本/job-admin/backend/project_paths.py) 已将所有运行时数据目录映射到根目录共享的 [dataset/](file:///D:/1/CS&AI/个人项目/ALL/job_system - 交付版本/dataset) 下。
  * `portrait_builder`（JD 抽取流水线）所用的上传路径（`uploads`）、跑批运行记录（`runs`）、临时快照（`snapshots`）和内部索引数据库都归集在 `dataset/runtime_data/portrait_builder_data/` 内，实现了真正的状态外部解耦，代码部分不保存任何运行时业务数据。

## 🔑 第二阶段：环境变量配置

在根目录下，将 `.env.example` 复制一份并重命名为 `.env`，完成以下核心模型配置：

* [ ] **向量模型配置 (Zhipu Embedding-3)**：
  配置用于标签治理、归一化、中英翻译和中文标签模糊搜索的向量化模型。
  ```ini
  JOB_SYSTEM_BIGMODEL_VECTOR_API_KEY=your_zhipu_api_key
  JOB_SYSTEM_VECTOR_PROFILE=bigmodel_embedding_3
  ```
* [ ] **大模型配置 (DeepSeek)**：
  配置用于简历解析和 JD 抽取 Pipeline 格式化任务的 LLM。
  ```ini
  # 学生端账户与基本 AI 处理
  CAREER_PLANNER_AI_LLM_BASE_URL=https://api.deepseek.com/v1
  CAREER_PLANNER_AI_LLM_API_KEY=your_deepseek_api_key
  # 推荐使用官网替代版 deepseek-v4-flash
  CAREER_PLANNER_AI_LLM_MODEL=deepseek-v4-flash 
  ```
* [ ] **JWT 密钥设置**：
  ```ini
  CAREER_PLANNER_JWT_SECRET=your_random_secret_string
  ```

## 🎯 第四阶段：核心业务功能自检清单

### 👥 学生客户端 (职途星 - Port 3000)
* [ ] **账号与登录**：注册新账号并成功登录。
* [ ] **AI 简历解析**：上传一份 PDF/Word 简历，确认系统能通过大模型将其解析为结构化标签，并填入个人画像页面。
* [ ] **标签语义搜索（词体验层）**：在“专业技能构建”搜索框中输入中文关键词（如“响应式”或“数据库”），系统应利用 **Zhipu Embedding-3** 召回关联的标准英文标签（如 `Responsive Design`、`Database`）。
* [ ] **多维人岗匹配**：确认能够在推荐面板中查看到分类为 **Safety (保底)**、**Target (精准)**、**Reach (挑战)** 的岗位列表，且可查看由 LLM 动态生成的匹配原因评语及技能差距报告。

### 🛡️ 管理后台端 (Job Admin - Port 5173)
* [ ] **岗位库 CRUD**：在后台管理面板中能够查看、添加、编辑和删除 [dataset/career.json](file:///D:/1/CS&AI/个人项目/ALL/job_system - 交付版本/dataset/career.json) 中的岗位记录。
* [ ] **JD 画像跑批 (Portrait Builder)**：
  * 上传一个包含原始招聘说明的 JD 文本，运行跑批任务。
  * 确认在跑批结束后，能自动提取出结构化的 `techStack`、`techCapabilities`（区分 `principle`/`engineering`/`scene`）及 `devTools` 属性。
* [ ] **标签治理归一 (Tag Normalization)**：
  * 在标签管理面板中，触发归一化治理，系统将利用 Zhipu Embedding 和大模型自动将非标中文词汇聚类并绑定到对应的标准英文 Tag 上。
  * 能够进入 **LLM 标签审核与复查面板**，逐条对系统自动归一的词条进行手动确认、修改或覆盖。
