## 一、系统总览与设计隐喻

### 1.1 核心隐喻：果园采摘（Orchard Harvest）

整个系统以**「果园采摘」**为设计隐喻，贯穿始终：

| 隐喻元素 | 系统对应 | 用户心智 |
|---|---|---|
| 🌳 果园 | 岗位推荐池 | "这是我的机会花园" |
| 🍎 果实（Slot） | 单个匹配岗位 | "每个果子都是一个机会" |
| ✅ 验熟（Check） | LLM 准入核查 | "看看这个果子我能不能摘" |
| 🧺 篮子（Basket） | 岗位收藏/提交单元 | "把想要的放进篮子" |
| 🌾 收割（Harvest） | 生成竞争力报告 | "看看我的收获" |
| 🍅 番茄钟（Action Plan） | 游戏化提升系统 | "我要让自己变得更强" |

### 1.2 设计基调

- **视觉风格**：清新插画风 + 数据可视化，兼具专业感与亲和力
- **色彩体系**：自然绿为主色（成长/生机），辅以暖橙（成熟/收获）、冷蓝（数据/理性）
- **交互原则**：渐进式披露（Progressive Disclosure）——先给结论，再给细节；先给推荐，再给分析

#### 3.1.3 Slot 卡片状态设计（四态）

| 状态 | 视觉表现 | 可用操作 |
|---|---|---|
| **🔒 Locked（未核查）** | 果实图标正常色，卡片有微弱脉冲动效提示"待核查" | 查看详情、点击 Check |
| **✅ Checked-Pass（可采摘）** | 果实图标发光，卡片边框变绿，出现「🧺 Pick 采摘」按钮 | 查看详情、Pick 到篮子 |
| **❌ Checked-Fail（不可采摘）** | 整卡灰度，果实图标加 💤，底部显示失败原因 | 仅查看详情（了解差距） |
| **🧺 Picked（已采摘）** | 卡片右上角出现篮子图标角标，Pick 按钮变为「已在篮子中」 | 查看详情、从篮子移除 |

#### 3.1.4 三种推荐路径的差异化展示

| 路径 | 卡片装饰 | 排序逻辑 | 特殊标注 |
|---|---|---|---|
| **🎯 精选推荐** | 默认绿色主题 | 按 Base Match Score 降序 | 显示"算法推荐"标签 |
| **💡 猜你喜欢** | 淡紫色点缀 | 按用户历史偏好 + 协同过滤 | 显示"基于你的偏好"标签 |
| **🔀 换岗推荐** | 橙色渐变装饰 | 按技能可迁移性评分 | 显示"跨领域机会"标签 + 迁移路径缩略图 |

### 📄 页面 2：篮子页（Basket）

#### 3.2.1 页面目标

管理当前正在编辑的篮子（唯一活跃篮子），对已 Pick 的岗位进行增删、预览对比，并最终提交进入报告生成流程。

#### 3.2.2 篮子规则说明

| 规则 | 说明 |
|---|---|
| **唯一活跃篮子** | 系统自动创建，用户不可手动新建。只有当前篮子被提交（进入 Ripening）后，系统才会自动生成下一个空篮子 |
| **自动创建时机** | 首次 Pick 时自动创建 / 上一个篮子提交后自动创建 |
| **编辑记录** | 每次增删操作自动记录时间戳，底部显示"上次编辑：3分钟前" |

## 四、全局状态机定义

### 4.1 Basket 状态机

```
                    用户首次 Pick
                        │
                        ▼
              ┌──────────────────┐
              │   📝 Drafting     │ ◄─── 可增删岗位
              │   (编辑中)        │ ◄─── 有且仅有 1 个活跃
              └────────┬─────────┘
                       │ 用户点击「提交篮子」
                       ▼
              ┌──────────────────┐
              │   ⏳ Queueing     │ ◄─── 等待系统资源
              │   (排队中)        │ ◄─── 不可编辑
              └────────┬─────────┘
                       │ 系统开始处理
                       ▼
              ┌──────────────────┐
              │   🔄 Ripening     │ ◄─── LLM 逐岗位生成报告
              │   (成熟中)        │ ◄─── 显示进度条
              └────────┬─────────┘
                       │ 全部岗位报告生成完毕
                       ▼
              ┌──────────────────┐
              │   🌾 Harvested    │ ◄─── 查看完整报告
              │   (已收割)        │ ◄─── 可选定目标岗位
              └──────────────────┘
                       │
                       ▼
            系统自动创建新的 Draft 篮子
```

### 4.2 Slot（岗位果实）状态机

```
          系统推荐生成
               │
               ▼
       ┌───────────────┐
       │  🔒 Locked     │ ◄─── 可查看详情
       │  (待核查)      │ ◄─── 可点击 Check
       └───────┬───────┘
               │ 点击 Check
               ▼
       ┌───────────────┐         ┌───────────────┐
       │  ✅ Pickable   │         │  ❌ Rejected   │
       │  (可采摘)      │         │  (不可采摘)     │
       └───────┬───────┘         └───────────────┘
               │ 点击 Pick                  │
               ▼                     可查看差距原因
       ┌───────────────┐          可跳转相似岗位
       │  🧺 Picked     │
       │  (已入篮)      │
       └───────┬───────┘
               │ 篮子提交后
               ▼
       ┌───────────────┐
       │  📊 Ranked     │ ◄─── 已生成报告
       │  (已排名)      │ ◄─── 可查看深度报告
       └───────┬───────┘
               │ 用户选定为目标
               ▼
       ┌───────────────┐
       │  🎯 Targeted   │ ◄─── 进入行动计划
       │  (已锁定目标)   │ ◄─── 生成提升任务
       └───────────────┘
```

### 5.1 动效规范

| 场景 | 动效类型 | 时长 | 说明 |
|---|---|---|---|
| Slot 卡片出现 | 从下向上渐入（staggered） | 每张间隔 80ms | 推荐列表加载时的果实"生长"感 |
| Check 通过 | 卡片边框绿色脉冲 + 果实发光 | 600ms | 传达"成熟了可以摘"的隐喻 |
| Check 失败 | 卡片灰度化 + 轻微震颤 | 400ms | 传达"这个果子还不适合" |
| Pick 入篮 | 果实缩小飞向底部 Tab 篮子图标 | 500ms 贝塞尔曲线 | 模拟放入购物车动效 |
| 篮子提交 | 篮子图标旋转 + 变为橙色（Ripening） | 800ms | 进入"成熟"等待状态 |
| 报告生成完毕 | 篮子图标金色闪光 + 角标提示 | 1000ms | 收获完成的仪式感 |
| 打卡确认 | 热力图格子从灰变亮 + 火焰 emoji 弹出 | 600ms | 多邻国式正向反馈 |
| 成长值增加 | 数字滚动上涨 + 进度条推进 | 800ms | 量化成长的可感知 |
| 段位升级 | 全屏庆祝动效 + 新段位图标展示 | 2000ms | 里程碑仪式感 |

## 七、预期效果总结

### 7.1 用户价值预期

| 维度 | 预期效果 |
|---|---|
| **认知降本** | 用户无需逐个分析 JD，系统自动对比并给出可视化的差距分析 |
| **决策提效** | 从"海投盲选"变为"精准对标"，篮子内横向对比让选择有理有据 |
| **行动驱动** | 不止于"告诉你缺什么"，更给出"怎么补"的完整行动路径 |
| **成长可见** | 打卡热力图 + 成长值 + 画像动态更新，让每一天的努力都可量化 |
| **信任建立** | 竞争力评分的可解释性（置信度函数、JD 逐项打分）让用户理解并信任系统 |

### 7.2 系统闭环

```
  ┌────────────────────────────────────────────────────┐
  │                                                    │
  │    画像构建 ──→ 智能推荐 ──→ 准入核查               │
  │       ↑                         │                  │
  │       │                         ▼                  │
  │    画像更新 ←── 行动提升 ←── 差距分析               │
  │                                                    │
  │           完整的「诊断-规划-执行-反馈」闭环            │
  │                                                    │
  └────────────────────────────────────────────────────┘
```

用户在系统内的每一次操作（浏览、Check、Pick、打卡、添加事件）都在强化画像精度，画像精度的提升又反向提高推荐和评分的准确性，形成**数据飞轮效应**。
