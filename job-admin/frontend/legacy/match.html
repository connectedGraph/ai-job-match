🧭 Demo 输入说明

本页是学生人岗匹配 Demo：左侧填入学生画像 JSON，点击后会调用后端 /api/match，按保守、精准、冲刺三档返回岗位。这里不是 mock 匹配，后端服务不可用时会直接报错。

核心画像字段
student_id string，学生唯一标识
direction string，求职方向
techStack[] { name, level }，具体可安装/可调用技术
techCapabilities[] { name, level, type }，抽象技术能力
devTools[] { name, level }，工具/平台/工程协作能力
softQuality[] 固定软素质维度，level 为 1-4
growthPotential[] 固定成长潜力维度，level 为 1-4
techCapabilities.type 分类
engineering 工程实践能力：Hybrid 架构、性能优化、工具链开发、工程化、组件设计。
scene 场景经验能力：跨端开发、特定业务/平台实战。
principle 基础原理/方法论：设计模式、架构方法、底层理论。

注意：后端评分会把 techStack、techCapabilities、devTools 聚合为“技术要求”80% 权重；type 主要用于技术能力归因和后续分类治理，普通技术能力不要写 soft_flag。
